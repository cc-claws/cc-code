# Markdown 表格第一列中文多时断行过窄，显示为竖排

**状态**：Fixed
**修复日期**：2026-05-18
**优先级**：中
**创建日期**：2026-05-18

## 问题描述

3-4 列的 markdown 表格在 TUI 中渲染时，如果第一列包含较多中文字符，按比例缩放的列宽算法会把所有列按比例压窄。当列数多但内容区宽度有限时，窄列宽度可能只有 3-8 显示列，对 CJK 字符（宽度 2）来说每行只能放 1-2 个字，形成不可读的竖排。

## 症状详情

- 表格列数 ≥ 3 时必现
- 第一列中文较多时，最终分配的列宽只有 4-8 个显示列
- CJK 字符每行显示 1-3 字后强制断行，竖排形态无法阅读
- 纯 ASCII 列在同一缩放比例下仍然可读（字符宽度 1），问题只影响 CJK

## 涉及文件

- `peri-widgets/src/markdown/render_state.rs` —— `TableBuilder::wrap_cells()`（L60-100）中 `scale_col_widths()` 纯比例缩放，无每列最小可读宽度下限

## 期望改进方向

`scale_col_widths()` 或 `wrap_cells()` 需要保证每列有合理的最小宽度（至少 6-8 显示列），当前 `max(1)` 下限太低。
