与 Claude Code (TS) 的设计差异
Architecture Differences
1. Panel vs Dialog
Claude Code: BackgroundTasksDialog 是 modal overlay（弹窗覆盖在聊天区上方）
cc-code: BackgroundTasksPanel 是 PanelComponent（替换屏幕下半部分，聊天区被压缩）
原因：peri 的 PanelManager 只支持同时打开 1 个 panel，无 overlay 层级
2. Ctrl+B → 输出 channel 切换（进程不中断）
Claude Code: shellCommand.background() 断开 UI 监听，进程继续运行，输出转存 DiskTaskOutput
cc-code: output_rx channel 切换 进程不中断，输出从 UI 渲染切换到 DiskOutputWriter 写磁盘
流式输出改造（execute_shell_command_streaming）是前置依赖，stdout/stderr 通过 mpsc channel 推送
3. ↓ to view 条件
Claude Code: · ↓ to view 仅在单个 ultraplan remote agent + phase=needs_input/plan_ready 时显示
cc-code: 有后台任务时始终显示 · ↓ to view（更简单，但信息密度略低）
原因：peri 暂无 ultraplan 概念，简化为始终可访问
4. 输出框边框
Claude Code: borderStyle="round"（Ink 原生圆角边框）
cc-code: Block::bordered().border_type(BorderType::Rounded)（ratatui >= 0.26 支持）
视觉一致，实现方式不同
5. 通知格式
Claude Code: <task-notification> XML 标签包裹（结构化，agent 可解析）
cc-code: [Background Task Completed] 纯文本前缀（简单，但 agent 需要正则解析）
建议后续改为结构化格式（JSON 或 XML），便于 agent 精确解析
