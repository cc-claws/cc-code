# 修复 thinking 渲染换行后缺少前缀缩进

## 问题

`message_render.rs:574-585` 给每个 markdown 逻辑行加 4 列前缀（`  ⎿ ` / `    `）。但 `parse_markdown` 的 `max_width` 只影响表格列宽，**不影响普通段落**——pulldown-cmark 不做 word wrap。长段落返回超长 `Line`，加前缀后超过终端宽度，`Paragraph::wrap` 折行后续行顶到最左边，没有缩进。

## 方案对比

| | A. wrap_line_spans | B. 纯文本 wrap | C. render_state 预折行 |
|---|---|---|---|
| 代码量 | ~45 行 | ~15 行 | ~20 行 |
| 保留 span 样式 | ✅ 完整 | ❌ 丢失高亮色相 | ✅ |
| 影响范围 | 仅 reasoning | 仅 reasoning | **所有 markdown** |
| 风险 | 低 | 极低 | 中（需全量回归） |
| 可单元测试 | ✅ | ✅ | 难 |

**推荐 A**。`dim_markdown_lines`（:15-37）刻意保留语法高亮色相（有 fg 的 span → 保留色相 + DIM 修饰），说明设计意图要保留高亮。B 方案丢失高亮虽实际影响小但违反既有设计。C 方案最优雅但影响面太大。

## 推荐方案：A — wrap_line_spans

### 安全性不变式

```
wrap_line_spans(line, content_width)  →  每行 ≤ content_width
加 4 列前缀后                          →  每行 ≤ content_width + 4 = width
build_wrap_map(lines, width)           →  line_count = 1，1:1 不二次折行
```

`wrap_map` / `viewport_clip`（二分裁剪）/ 选区高亮（`visual_to_logical`）/ 滚动条 **全部自动正确**，零改动。

VM 层 `width` 与渲染线程 `self.width` 同源（`RenderTask` 内 `messages_to_view_models(width)` 和 `build_wrap_map(lines, width)` 用同一个 width），保证一致。

### 实现位置

`peri-tui/src/ui/message_render.rs`，私有函数。

### 完整实现代码

```rust
use unicode_width::UnicodeWidthChar;

/// 将一个含多 span 的 Line 按视觉宽度折行，保留各 span 样式。
///
/// 算法：flatten → 贪心宽度折行（单词边界优先）→ reassemble。
/// 用于 reasoning 渲染：确保每行宽度 ≤ max_width，加前缀后不触发 Paragraph::wrap 二次折行。
fn wrap_line_spans(line: Line<'static>, max_width: usize) -> Vec<Line<'static>> {
    // [0] 空行或 max_width=0 直接返回
    if max_width == 0 || line.spans.is_empty() {
        return vec![line];
    }

    // [1] flatten: spans → Vec<(char, Style)>
    let flat: Vec<(char, Style)> = line
        .spans
        .iter()
        .flat_map(|s| s.content.chars().map(move |c| (c, s.style)))
        .collect();

    // [快速路径] 不超宽原样返回
    let total_width: usize = flat.iter().map(|(c, _)| c.width().unwrap_or(0)).sum();
    if total_width <= max_width {
        return vec![line];
    }

    // [2] 贪心折行
    let mut result: Vec<Line<'static>> = Vec::new();
    let mut pos = 0;
    while pos < flat.len() {
        // 累积宽度
        let mut cur_width = 0usize;
        let mut content_end = pos;
        for i in pos..flat.len() {
            let cw = flat[i].0.width().unwrap_or(0);
            if content_end > pos && cur_width + cw > max_width {
                break;
            }
            cur_width += cw;
            content_end = i + 1;
        }

        // 单词边界优先：往回找最后一个空白
        let mut break_at = content_end;
        for i in (pos..content_end).rev() {
            if flat[i].0.is_whitespace() {
                break_at = i;
                break;
            }
        }

        // trim 行首行尾空白
        let mut seg_start = pos;
        while seg_start < break_at && flat[seg_start].0.is_whitespace() {
            seg_start += 1;
        }
        let mut seg_end = break_at;
        while seg_end > seg_start && flat[seg_end - 1].0.is_whitespace() {
            seg_end -= 1;
        }

        // [3] reassemble: 相邻同 Style 字符合并
        if seg_start < seg_end {
            let mut spans: Vec<Span<'static>> = Vec::new();
            let mut cur_text = String::new();
            let mut cur_style = flat[seg_start].1;
            for &(ch, st) in &flat[seg_start..seg_end] {
                if st == cur_style {
                    cur_text.push(ch);
                } else {
                    spans.push(Span::styled(
                        std::mem::take(&mut cur_text),
                        cur_style,
                    ));
                    cur_text = ch.to_string();
                    cur_style = st;
                }
            }
            if !cur_text.is_empty() {
                spans.push(Span::styled(cur_text, cur_style));
            }
            result.push(Line::from(spans));
        }

        // 推进 pos，跳过断行点后的连续空白
        pos = break_at;
        while pos < flat.len() && flat[pos].0.is_whitespace() {
            pos += 1;
        }
    }

    if result.is_empty() {
        vec![Line::default()]
    } else {
        result
    }
}
```

### 边界条件

| 场景 | 处理 |
|------|------|
| 空行 | 返回 `vec![Line::default()]` |
| 单 char 宽度 > max_width（极窄终端） | 强制放入（`content_end > pos` 在 break 判断后推进） |
| CJK 无空格长串 | 硬断，逐字推进 |
| 中英混排 `你好 hello world` | CJK 部分逐字断到宽度用尽，ASCII 部分在空格断 |
| span 跨越断行点 | flatten 已消除 span 边界，reassemble 重新合并 |
| 零宽字符（ZWJ 等） | `width().unwrap_or(0)` = 0，不触发 break，随相邻字符走 |

### 调用点改动（message_render.rs:579-585）

```rust
// 改前
for (i, mut line) in dimmed.into_iter().enumerate() {
    let prefix = if i == 0 { "  ⎿ " } else { "    " };
    let mut spans = vec![Span::styled(prefix, Style::default().fg(theme::DIM))];
    spans.append(&mut line.spans);
    lines.push(Line::from(spans));
}

// 改后
for (i, line) in dimmed.into_iter().enumerate() {
    for (j, wline) in wrap_line_spans(line, content_width).into_iter().enumerate() {
        let prefix = if i == 0 && j == 0 { "  ⎿ " } else { "    " };
        let mut spans = vec![Span::styled(prefix, Style::default().fg(theme::DIM))];
        spans.extend(wline.spans);
        lines.push(Line::from(spans));
    }
}
```

关键：`i == 0 && j == 0` — 只有第一个逻辑行的第一个视觉行用 `⎿`。

## 不受影响

- `dim_markdown_lines`：签名不变，在 wrap 前调用
- `parse_markdown` / `MarkdownCache`：不变
- `build_wrap_map` / `viewport_clip` / 选区高亮：自动适配（预折行后 1:1）
- UserBubble / ToolBlock / AssistantBubble Text block：不经过 `wrap_line_spans`

## 性能分析

| 指标 | 分析 |
|------|------|
| 时间复杂度 | O(n)，n = 字符数。flatten + 单次遍历折行 + reassemble |
| 空间复杂度 | O(n)，flat Vec 临时分配 |
| 典型 reasoning 长度 | 200-2000 字符，耗时 < 0.1ms |
| 调用频率 | 每次 RebuildAll 对 reasoning 行调用一次；RebuildAll 有 16ms 节流 |
| MarkdownCache 影响 | 无——wrap 在缓存之后执行，不经过 LRU 缓存 |
| 快速路径 | 90%+ 的行不超宽（短段落、列表项），直接返回原 line，零开销 |
